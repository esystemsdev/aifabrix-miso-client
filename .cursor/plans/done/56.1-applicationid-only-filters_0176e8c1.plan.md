# applicationId-only input migration plan

## Scope and decision

- Enforce `applicationId` as the only application selector for log API **inputs**.
- Apply to all log input surfaces, including jobs, stats, and export.
- Keep `application` in **response** models for compatibility (read-only compatibility mode).
- Out of scope: non-logs domains where `application` appears (`auth`, `roles`, `permissions`, config/runtime context fields).
